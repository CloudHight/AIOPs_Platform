# **03_deploy.py - Line-by-Line Explanation**

This script deploys a trained Random Cut Forest model to a **SageMaker endpoint** so it can serve predictions (anomaly scores) in real-time. Here's what each part does:

---

## **1. Import Libraries**
```python
import json
from sagemaker import RandomCutForest
```
- **json**: For reading/writing JSON files that store model/endpoint info
- **RandomCutForest**: SageMaker's RCF class (to interact with trained model)

---

## **2. Function Definition - deploy_model()**
```python
def deploy_model(model_info_file="model_info.json",
                 endpoint_info_file="endpoint_info.json",
                 instance_count=1,
                 instance_type="ml.t2.medium"):
```
Defines the main deployment function with these parameters:
- `model_info_file`: JSON file with training job info (created by `02_train.py`)
- `endpoint_info_file`: Where to save endpoint info (for later use/cleanup)
- `instance_count`: How many instances to deploy (default: 1)
- `instance_type`: AWS instance type for hosting (default: `"ml.t2.medium"` - small, cheap)

---

## **3. Print Deployment Header**
```python
    print("========== Model Deployment ==========")
```
- Simple visual separator in logs

---

## **4. Load Model Information**
```python
    # Load model info
    with open(model_info_file, 'r') as f:
        model_info = json.load(f)
```
- Opens `model_info.json` (created during training)
- Loads the JSON content into a Python dictionary
- **Example content**: `{"training_job_name": "randomcutforest-2025-01-01-12-00-00-123"}`

---

## **5. Attach to Trained Model**
```python
    # Attach to existing model
    rcf = RandomCutForest.attach(model_info['training_job_name'])
```
- **Key concept**: The model was already trained and saved to S3 by `02_train.py`
- `.attach()`: Connects to the **existing** trained model in SageMaker using its job name
- Does **NOT** retrain the model - just creates a reference to the trained artifacts
- `rcf` object now represents the trained model ready for deployment

---

## **6. Deploy Model to Endpoint**
```python
    predictor = rcf.deploy(
        initial_instance_count=instance_count,
        instance_type=instance_type
    )
```
- **`.deploy()`**: The core deployment action
- SageMaker does several things:
  1. Creates a **REST API endpoint** (URL)
  2. Spins up the specified instance(s) (`ml.t2.medium`)
  3. Loads the trained model onto the instance(s)
  4. Sets up autoscaling, monitoring, etc.
- Returns a `predictor` object that can make predictions against the endpoint
- **Note**: This can take 2-5 minutes as AWS provisions resources

---

## **7. Save Endpoint Information**
```python
    # Save endpoint info
    endpoint_info = {'endpoint_name': predictor.endpoint_name}
    with open(endpoint_info_file, 'w') as f:
        json.dump(endpoint_info, f)
```
- Creates a dictionary with just the endpoint name
- Saves to `endpoint_info.json` for:
  - Later scripts to know which endpoint to call
  - Cleanup script to know which endpoint to delete
- **Example content**: `{"endpoint_name": "randomcutforest-2025-01-01-12-34-56-789"}`

---

## **8. Print Success Message and Return**
```python
    print(f"Model deployed to endpoint: {predictor.endpoint_name}")
    print(f"Endpoint info saved to {endpoint_info_file}")
    return predictor
```
- Prints the actual endpoint name (auto-generated by SageMaker)
- Confirms where endpoint info was saved
- Returns the `predictor` object for use in other scripts

---

## **9. Function Definition - cleanup_endpoint()**
```python
def cleanup_endpoint(endpoint_info_file="endpoint_info.json"):
```
- Defines cleanup function to **delete the endpoint** (to avoid AWS charges)
- Takes the endpoint info file as parameter

---

## **10. Delete Endpoint Logic**
```python
    """Clean up the deployed endpoint to avoid charges."""
    try:
        import boto3
        with open(endpoint_info_file, 'r') as f:
            endpoint_info = json.load(f)
        
        client = boto3.client('sagemaker')
        client.delete_endpoint(EndpointName=endpoint_info['endpoint_name'])
        print("Endpoint deleted successfully")
    except Exception as e:
        print(f"Error deleting endpoint: {e}")
```
1. **Try block**: Safeguards against errors
2. **Import boto3**: AWS SDK for Python (only imported when needed)
3. **Load endpoint info**: Reads the endpoint name from JSON
4. **Create SageMaker client**: Connection to AWS SageMaker API
5. **Delete endpoint**: Calls AWS API to terminate the endpoint
6. **Exception handling**: If anything fails (file missing, permissions, etc.), prints error

**IMPORTANT**: Endpoints cost money while running! Always delete when done.

---

## **11. Main Execution Block**
```python
if __name__ == "__main__":
    deploy_model()
```
- When script runs directly, deploys the model
- Allows script to be imported OR run standalone

## **KEY CONCEPTS EXPLAINED**

### **What is a SageMaker Endpoint?**
- A **web service** (REST API) that hosts your ML model
- URL: `https://runtime.sagemaker.us-east-1.amazonaws.com/endpoints/your-endpoint-name/invocations`
- Accepts data → Returns predictions
- Managed by AWS (scaling, monitoring, security)

### **Why Deploy?**
- **Before**: Model exists as files in S3 (can't make predictions)
- **After**: Model is live on a server (can make predictions via API)

### **Cost Implications**
- `ml.t2.medium`: ~$0.05-$0.10 per hour
- **ALWAYS DELETE** when not using to avoid unexpected charges
- Endpoint runs 24/7 until deleted

### **Real-World Analogy**
Think of it like:
- **Training**: Baking a cake (model) and putting it in the fridge (S3)
- **Deployment**: Taking the cake out, putting it in a display case (endpoint) so customers can buy slices (predictions)
- **Cleanup**: Closing the shop and putting the cake away
